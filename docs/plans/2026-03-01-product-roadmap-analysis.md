## 1. 執行摘要

### 現狀

Rexiano 完成 Phase 1-6，已具備完整的下落音符視覺化、MIDI 裝置連接、音頻播放、練習模式（Wait/Watch/Free）、速度控制、A-B 循環、評分系統等核心功能。Phase 6.5 的兒童可用性增強也大致完成（琴鍵音名、暗色主題、拖放匯入、設定面板、節拍器、新手引導、18 首內建曲庫等）。

### 競品格局

| 產品 | 定位 | 價格 | 核心優勢 |
|------|------|------|----------|
| **Synthesia** | MIDI 視覺化學習 | $39 一次性 | 落鍵視覺、廣泛 MIDI 支援、指法建議 |
| **Simply Piano** | 結構化課程 | ~$170/年 | 遊戲化課程、5000+ 歌曲、即時反饋 |
| **Flowkey** | 歌曲導向學習 | ~$120/年 | 1500+ 歌曲分級、Wait Mode、影片教學 |
| **Piano Marvel** | 學術式評量 | ~$120/年 | SASR 視奏評量、譜面教學、Hal Leonard 授權樂譜 |
| **MuseFlow** | AI 視奏訓練 | ~$100/年 | AI 生成不重複樂譜、強迫視奏（不可背譜） |

### 機會

Rexiano 的獨特優勢在於：**開源免費 + 五線譜顯示（Phase 7）+ 多主題 + 離線運作 + 完全可自訂**。在開源生態中，沒有任何競品同時具備五線譜、下落音符、練習模式三大功能。現有的開源替代品（Neothesia、PyPiano、Melodigram）大多只有基本的落鍵視覺化，缺乏練習和評分功能。

### 建議策略

1. **短期（Phase 7）**：完成五線譜顯示，這是對 Synthesia 最大的差異化武器
2. **中期（Phase 7.5）**：加入結構化課程系統 + 成就系統，這是 Simply Piano 的核心吸引力
3. **長期（Phase 8+）**：視奏訓練 + AI 輔助，這是下一代鋼琴教學 app 的方向

## 3. 兒童鋼琴教育研究

### 3.1 六歲兒童的認知特點

根據兒童鋼琴教育的研究與實踐經驗：

- **短期記憶容量有限**：6 歲兒童一次只能記住 2-3 個項目（成人可記 7-9 個），因此每次練習的目標不宜超過 2-3 個重點
- **注意力持續時間短**：約 10-15 分鐘，需要頻繁切換活動類型
- **以遊戲為最佳學習方式**：「gamify everything」——所有年齡的孩子透過玩來學習最有效
- **需要即時正向回饋**：每一個小成功都應被看見和慶祝
- **家長參與至關重要**：這個年齡的孩子需要家長坐在旁邊引導練習

### 3.2 遊戲化最佳實踐

#### 有效的遊戲化元素

| 元素 | 效果 | 實證 |
|------|------|------|
| **貼紙/徽章收集** | 短期動機極強 | 鋼琴教師普遍使用，研究顯示可建立練習習慣 |
| **等級/進度條** | 提供明確的前進感 | Simply Piano 的等級系統使次日留存率從 13% 提升至 55% |
| **連擊（Streak）** | 鼓勵持續正確 | Rexiano 已實作 combo 系統 |
| **解鎖內容** | 創造期待感 | 新歌曲、新主題作為獎勵 |
| **每日目標** | 建立規律 | 「今天練 10 分鐘」比「把這首練好」更可行 |

#### 需避免的陷阱

| 陷阱 | 問題 | 解法 |
|------|------|------|
| 過度依賴外在獎勵 | 移除獎勵後動機消失 | 獎勵系統是橋樑，最終目標是建立內在動機（享受音樂本身的樂趣） |
| 懲罰性回饋 | 「錯了！」讓孩子沮喪 | 永遠用鼓勵語氣：「差一點點！再試一次？」 |
| 過長的練習段落 | 注意力不集中 | 5 分鐘一個活動，多個活動組成一次練習 |
| 缺乏選擇權 | 被動學習 | 讓孩子選擇下一首要練的曲子 |

### 3.3 練習動機維持策略

根據鋼琴教師實務經驗整理：

1. **短期循環（每日）**：設定一個小小的今日目標 → 完成 → 即時獎勵（數位貼紙 / 星星）
2. **中期循環（每週）**：累積一週的練習 → 解鎖新曲目或新主題色
3. **長期循環（每月）**：完成一個「關卡」→ 里程碑慶祝 → 家長報告（「Rex 這個月學了 5 首新曲子！」）
4. **社交認可**：錄製表演並分享（對於家長來說非常重要的功能）

### 3.4 對 Rexiano 的啟示

| 教育需求 | 目前 Rexiano 的覆蓋 | 建議行動 |
|----------|---------------------|---------|
| 短注意力 → 短練習段落 | 部分（A-B 循環） | 新增「每日挑戰」模式（5 分鐘微練習） |
| 即時正向回饋 | ✅（flashHit、combo、慶祝畫面） | 可加強（音效 + 視覺 + 語音鼓勵） |
| 進度可見化 | 部分（成績持久化） | 新增成就系統 + 練習日曆 |
| 家長參與 | ❌ | 新增家長儀表板 / 練習報告 |
| 自主選擇 | ✅（曲庫選擇） | 可加強（推薦下一首適合的曲子） |
| 遊戲化 | 部分（combo、慶祝） | 新增徽章系統 + 每日目標 |

## 5. 各功能一頁概要

### 功能 1：五線譜顯示

> 詳細技術設計見 `docs/plans/2026-03-01-phase7-sheet-music-design.md`

**價值**：高 — 這是 Rexiano 對 Synthesia 最大的差異化。Synthesia 不支援五線譜，Simply Piano / Flowkey 不支援下落音符。Rexiano 將成為唯一同時提供兩種視覺模式的開源工具。對 Rex 而言，初期使用下落音符學習，逐漸過渡到五線譜閱讀，是完美的學習路徑。

**複雜度**：極高 — MIDI → 五線譜的量化轉換是核心難題。

**設計概念**：三種顯示模式（雙面板 / 僅譜面 / 僅落鍵），播放游標跟隨，當前音符高亮。

---

### 功能 2：成就/徽章系統

**價值**：高 — 對 6 歲兒童，收集徽章是極強的外在動機。研究顯示這類遊戲化元素可顯著提高練習頻率和持續性。

**複雜度**：低 — 基於現有的 `useProgressStore` + `SessionRecord` 即可實現。

**設計概念**：

```typescript
interface Achievement {
  id: string;
  title: string;          // "初次演奏"、"速度大師"
  description: string;    // "完成第一首曲目"
  icon: string;           // emoji 或 SVG
  category: 'milestone' | 'streak' | 'accuracy' | 'exploration';
  condition: (sessions: SessionRecord[]) => boolean;
  unlockedAt?: number;    // Unix timestamp
}

// 範例徽章：
const ACHIEVEMENTS: Achievement[] = [
  { id: 'first_play',    title: '初次演奏',   condition: s => s.length >= 1 },
  { id: 'perfect_score', title: '完美演出',   condition: s => s.some(r => r.score.accuracy >= 95) },
  { id: '7day_streak',   title: '七日連續',   condition: s => hasConsecutiveDays(s, 7) },
  { id: 'speed_demon',   title: '速度大師',   condition: s => s.some(r => r.speed >= 1.5 && r.score.accuracy >= 80) },
  { id: '10_songs',      title: '十曲通關',   condition: s => uniqueSongs(s) >= 10 },
  { id: 'both_hands',    title: '雙手協奏',   condition: s => s.some(r => r.tracksPlayed.length >= 2) },
  // ... 更多
];
```

**UI**：設定面板或主畫面新增「成就櫥窗」，已解鎖的徽章金色發光，未解鎖的灰色剪影。新徽章解鎖時全屏彈出 + 粒子效果。

**估計工時**：2-3 天（Store + UI + 條件判斷）

---

### 功能 3：練習日曆 + 每日目標

**價值**：高 — 建立規律的練習習慣是 6 歲兒童學琴的最大挑戰。視覺化的日曆讓進度一目了然，對家長也很有用。

**複雜度**：低 — 資料已存在（`SessionRecord.timestamp`），只需新增 UI 和每日目標邏輯。

**設計概念**：

```
┌─────────────────────────────┐
│  三月 2026                   │
│  一  二  三  四  五  六  日  │
│                  1   2   3   │
│  ⭐ ⭐ ⭐ ⭐  5   6   7   │  ← 星星 = 練習日
│  ⭐ ⭐ 10  11  12  ⭐ ⭐  │
│  ...                         │
│                               │
│  今日目標：練習 10 分鐘       │
│  ████████████░░░░  7/10 分鐘  │
│                               │
│  🔥 連續 4 天！               │
└─────────────────────────────┘
```

- 每日目標：預設 10 分鐘（可在設定中調整：5/10/15/20/30 分鐘）
- 連續天數（streak）：連續完成每日目標的天數，強調不間斷
- 完成每日目標時觸發小慶祝動畫
- 資料來源：`useProgressStore.sessions` 按日期聚合

**估計工時**：3-4 天

**估計工時**：5-7 天

### 功能 5：智慧曲目推薦

**價值**：中 — 避免孩子總是練同一首歌或挑太難的歌。

**複雜度**：中 — 需要建立推薦邏輯，但不需要 ML。

**設計概念**：

```typescript
/**
 * 推薦演算法（啟發式，非 ML）：
 *
 * 1. 過濾出難度適合的曲目
 *    - 如果最近幾首 accuracy > 85% → 可以推薦稍難的
 *    - 如果最近 accuracy < 60% → 推薦同難度或更簡單的
 *
 * 2. 優先推薦：
 *    a. 從未練過的曲目（探索新曲）
 *    b. 練過但 accuracy < 70% 的曲目（需要複習）
 *    c. 最近沒練過（> 7 天）但之前練得不錯的（維持記憶）
 *
 * 3. 排除最近 3 次已練過的曲目（避免重複疲勞）
 */
function recommendNextSong(
  sessions: SessionRecord[],
  allSongs: BuiltinSongMeta[]
): BuiltinSongMeta[] { ... }
```

- 在 SongLibrary 頁面頂部顯示「推薦給你」section（2-3 首）
- 附上推薦理由：「你還沒試過這首！」「這首可以再練練，上次 68%」

**估計工時**：3-4 天

---

### 功能 6：結構化課程系統

**價值**：高 — Simply Piano 和 Piano Marvel 的核心吸引力。對 Rex 這樣的初學者，有引導的學習路徑比自由探索更有效。

**複雜度**：高 — 需要課程內容設計（非純技術問題）+ 新的 UI 流程。

**設計概念**：

```typescript
interface Lesson {
  id: string;
  title: string;              // "認識中央 C"
  description: string;
  prerequisite?: string;       // 前一課的 ID
  type: 'concept' | 'exercise' | 'song';
  content: LessonContent;     // 因 type 而異
  completionCriteria: {
    minAccuracy?: number;      // 最低準確率（如 70%）
    minSpeed?: number;         // 最低速度（如 0.5x）
  };
}

interface Course {
  id: string;
  title: string;               // "初學者之路"
  lessons: Lesson[];
  targetAge: 'child' | 'teen' | 'adult';
}

// 範例課程：
const BEGINNER_COURSE: Course = {
  id: 'beginner',
  title: '初學者之路',
  lessons: [
    { id: 'l1', title: '認識鍵盤',     type: 'concept',  content: { ... } },
    { id: 'l2', title: '找到中央 C',    type: 'exercise', content: { targetNote: 60 } },
    { id: 'l3', title: 'C-D-E 三個音',  type: 'exercise', content: { notes: [60,62,64] } },
    { id: 'l4', title: '第一首歌！',     type: 'song',     content: { songId: 'hot-cross-buns' } },
    // ...
  ],
};
```

- 初期可從內建 18 首曲庫挑選 + 新增專門的練習片段
- 課程內容以 JSON 定義，不硬編碼
- 未來可讓社群貢獻課程（PR 到 `resources/courses/` 目錄）

**估計工時**：10-15 天（含內容設計）

---

### 功能 7：視奏訓練模式

**價值**：中 — 對 Rex 目前的程度略為超前，但是長期非常重要的技能。MuseFlow 證明了視奏訓練是下一代鋼琴教學 app 的核心差異化。

**複雜度**：高 — 需要即時生成樂譜 + 與 Phase 7 五線譜整合。

**設計概念**：

- 根據使用者程度自動生成簡單的旋律片段（4-8 小節）
- 使用規則生成（非 AI）：限定音域、限定節奏型態、限定調號
- 倒計時後開始，不可暫停（模擬真實視奏）
- 結束後評分 + 建議
- 需要 Phase 7 五線譜作為前置

**估計工時**：7-10 天

---

### 功能 8：練習錄音 + 回放

**價值**：中 — 讓孩子聽自己彈的版本 vs 原版，培養自我評估能力。家長也可以遠端「聽到」孩子的練習。

**複雜度**：中 — Web Audio API 有 `MediaRecorder`，但需要處理 MIDI 錄製 + 音頻同步。

**設計概念**：

```typescript
interface PracticeRecording {
  id: string;
  songId: string;
  timestamp: number;
  // MIDI 事件序列（比音頻錄製更輕量）
  events: { time: number; type: 'noteOn' | 'noteOff'; midi: number; velocity: number }[];
  duration: number;
}
```

- 錄製 MIDI 事件（非音頻），檔案極小
- 回放時使用現有的 AudioEngine 重新合成
- 可與原曲對比播放（我的演奏 vs 原始 MIDI）

**估計工時**：5-7 天

---

### 功能 9：自訂曲庫擴充（社群分享）

**價值**：中 — 18 首內建曲庫對長期使用者不夠。

**複雜度**：中 — 需要線上曲庫倉庫設計。

**設計概念**：

- 在 GitHub 上建立 `rexiano-songbook` repository
- 使用者可透過 PR 貢獻 MIDI + 元資料（title, composer, difficulty, category）
- Rexiano 在設定中可「下載社群曲庫」（透過 GitHub API 列出 + 下載）
- 或者更簡單：提供「匯入 MIDI 資料夾」功能，讓使用者指定一個包含 MIDI 檔案的目錄

**估計工時**：5-7 天

---

### 功能 10：樂譜編輯器

> 已規劃在 Phase 8，詳見 DESIGN.md

**價值**：對 Rex 低（6 歲不需要編輯樂譜），但對開源社群有吸引力。

**複雜度**：極高 — Piano Roll 編輯器是完整的 DAW 子系統。

**建議**：維持 Phase 8 的規劃，但優先順序排在成就系統、課程系統之後。

---

### 功能 11：AI 輔助練習建議

**價值**：中 — 「這段你經常彈錯，建議放慢速度多練」。

**複雜度**：極高 — 需要分析 noteResults 歷史，可能需要 LLM 或 ML 模型。

**設計概念**（輕量版，不需 AI）：

- 基於規則的弱點分析：統計每個音符的 hit/miss 率
- 識別「問題段落」：連續 miss 的小節區間
- 生成建議：「第 8-12 小節的 miss 率為 45%，建議用 A-B 循環以 50% 速度練習」
- 進階版（需 LLM）：自然語言的個人化建議

**建議**：Phase 6.5 Sprint 5 的「練習洞察分析」已涵蓋基礎版。真正的 AI 建議列為長期目標。

---

### 功能 12-15 概要

| 功能 | 建議 |
|------|------|
| **多人/親子模式** | 技術上需要 split-screen MIDI routing，複雜度高。建議長期不列入，除非社群有強烈需求。 |
| **行動平台** | 需要完全重寫為 React Native 或 Flutter。投入產出比低。建議維持桌面平台定位。 |
| **MusicXML 匯入** | Phase 7 完成後，MusicXML 匯入可作為 Phase 8 的一部分。VexFlow 本身不直接支援，但轉換相對簡單。 |
| **音樂理論互動教學** | 如「什麼是和弦？」「什麼是大調音階？」的互動式教學卡片。需要大量教學內容設計。可作為課程系統的子集。 |

## 6. Rexiano 獨特定位

### 6.1 核心差異化

Rexiano 應定位為：**專為兒童設計的開源鋼琴學習工具，同時具備專業級的視覺化與譜面功能**。

```
                ┌─────────────────────────────────────────┐
                │           Rexiano 的獨特交集             │
                │                                         │
                │   ┌──────────┐   ┌──────────────────┐  │
                │   │ 開源免費  │ ∩ │ 五線譜 + 落鍵雙模式│  │
                │   └──────────┘   └──────────────────┘  │
                │         ∩                ∩              │
                │   ┌──────────────┐ ┌──────────────┐    │
                │   │ 兒童友善 UI  │ │ 完整練習模式  │    │
                │   └──────────────┘ └──────────────┘    │
                │                                         │
                │   沒有任何競品同時佔據這四個象限          │
                └─────────────────────────────────────────┘
```

### 6.2 五大差異化支柱

| 支柱 | 說明 | 競品對比 |
|------|------|---------|
| **1. 雙模式視覺化** | 下落音符 + 五線譜同步顯示，其他競品只有其一 | Synthesia 無五線譜；Simply Piano 無落鍵 |
| **2. 開源 + 免費** | 永遠免費，社群可貢獻課程和曲庫 | Synthesia $39；Simply Piano $170/年 |
| **3. 離線優先** | Electron 桌面 app，不依賴網路 | Flowkey / Piano Marvel 依賴雲端 |
| **4. 兒童友善但不幼稚** | 精心設計的 UI 主題（4 套），音名標籤，大按鈕 | Synthesia 界面老舊；Simply Piano 偏手機導向 |
| **5. 完全可自訂** | 自訂 MIDI、自訂 SoundFont、開源代碼 | 競品封閉，不可擴充 |

### 6.3 開源優勢的放大策略

| 策略 | 做法 |
|------|------|
| **社群曲庫** | 建立 `rexiano-songbook` repo，鼓勵使用者貢獻 MIDI + 難度標記 |
| **社群課程** | 讓鋼琴教師以 JSON 格式貢獻教學課程 |
| **Plugin 架構（長期）** | 讓開發者可以擴充新的練習模式或視覺效果 |
| **教育機構採用** | 免費 + 離線 + 可自訂 = 對學校和琴行有吸引力 |

### 6.4 家長功能的價值

對於 6 歲兒童的產品，**家長是真正的「購買決策者」**（即使 Rexiano 免費，安裝和設定仍由家長執行）。以下功能對家長至關重要：

1. **練習報告**：「我的孩子這週練了多久？進步了嗎？」
2. **成就系統**：「看，Rex 又拿到一個新徽章了！」（製造話題，鼓勵孩子）
3. **練習提醒**：桌面通知「該練琴了」（簡單的系統通知，不做 push notification）
4. **錄音回放**：「讓我聽聽你今天練的 Twinkle Twinkle」

這些功能技術複雜度不高，但對產品的「黏性」影響巨大。
