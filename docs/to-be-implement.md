請創建一個團隊，分成紅隊 (攻擊) 和 白隊 (防守) 互相競爭。這邊的攻擊是指根據主題，找出目前專案不完美的項目，而白隊則是根據這些項目修復，並主動找出進一步強化的地方。 經過 3 回合後，會經由裁判裁決是否還有未修復的地方，如果有且是紅隊曾經提出的問題，則紅隊勝 (因為紅隊已經找到該問題，但白隊未正確修復)。如果沒有問題，或是找出新的問題，則白隊勝 (因為白隊已經修復所有問題，或是紅隊未找出裁判發現的問題)。 另外嚴謹的話，白隊都必須要通過測試和 code review, 才算完成自己的回合。

這次的主題是 : "節奏" - 強化音符顯示和彈奏之間的拍點，節奏顯示，或是任何和彈奏時的體驗有關，皆是這次攻防項目。

紅白隊攻防主題提案
```
1. 「視覺回饋」
強化彈奏時的視覺反應品質 — Hit/Miss 閃光效果、連擊動畫、慶祝結算畫面、鍵盤高亮、音符狀態變化等所有與「使用者按下琴鍵後眼睛看到什麼」有關的視覺回饋，皆是這次攻防項目。包含動畫流暢度、時間感受、色彩辨識度、以及不同主題下的一致性。

2. 「五線譜」
五線譜顯示的正確性與可用性 — VexFlow 渲染的音符、休止符、調號、拍號、小節線、譜號分配、表情記號、播放同步游標、翻頁邏輯、Split 模式佈局，以及任何與「看譜學琴」體驗有關的項目，皆是這次攻防項目。重點在音樂理論正確性與視覺清晰度。

3. 「兒童體驗」
6 歲兒童獨立使用的順暢度 — 從啟動 App 到選曲、開始彈奏、完成練習的完整流程中，任何讓兒童困惑、需要大人協助、或操作不直覺的地方，皆是這次攻防項目。包含文字是否看得懂、按鈕是否夠大、流程是否太多步驟、錯誤訊息是否友善、以及 UI 縮放在不同螢幕的表現。

4. 「音訊品質」
彈奏時聽到的聲音品質與時間精確度 — SoundFont 音色、音符觸發延遲、AudioScheduler 排程精度、節拍器音效、錯誤音效、力度對應、音符釋放自然度、速度變化時的音高穩定性，以及任何與「耳朵聽到什麼」有關的體驗，皆是這次攻防項目。

5. 「效能與穩定性」
長時間使用下的流暢度與可靠性 — PixiJS 渲染 FPS、記憶體增長（音符池 / Text 池 / Sprite 洩漏）、大型 MIDI 檔案（1000+ 音符）的載入與播放、快速 seek 時的狀態一致性、MIDI 裝置熱插拔的穩健性，以及任何與「App 會不會卡頓或崩潰」有關的項目，皆是這次攻防項目。
```

每項比賽結束後的流程 : 
修復剩餘問題 --> full test --> code review --> merge to dev --> clean worktree --> next topic
